# 软考中级软件设计师考试大纲

**整理时间**: 2026-03-30  
**信息来源**: 全国计算机技术与软件专业技术资格（水平）考试官方大纲、历年真题

---

## 一、考试概述

### 1.1 基本信息

| 项目 | 内容 |
|------|------|
| **考试名称** | 计算机技术与软件专业技术资格（水平）考试 |
| **级别** | 中级 |
| **资格名称** | 软件设计师 |
| **证书效力** | 国家职业资格证书，以考代评 |
| **考试形式** | 笔试（无纸化机考） |
| **考试时间** | 每年 5 月、11 月各一次 |
| **合格标准** | 两科均≥45 分（满分 75 分） |

### 1.2 考试科目

| 科目 | 考试时间 | 题型 | 题量 | 满分 | 合格线 |
|------|---------|------|------|------|--------|
| **基础知识** | 150 分钟 | 选择题 | 75 题 | 75 分 | 45 分 |
| **应用技术** | 150 分钟 | 案例分析题 | 5-6 大题 | 75 分 | 45 分 |

**注意**：
- 两科必须**同时及格**才算通过
- 单科成绩不保留，需一次性通过两科
- 上午题和下午题在同一场考试中完成

### 1.3 考试目标

通过本考试的合格人员能根据软件开发项目管理和软件工程的要求，按照系统总体设计规格说明书进行软件设计；能指导程序员进行编码实现；具有工程师的实际工作能力和业务水平。

## 三、应用技术（下午题）

### 3.1 考试科目结构

| 题型 | 题量 | 分值 | 考查内容 |
|------|------|------|---------|
| 数据流图设计 | 1 题 | 15 分 | DFD 分析与设计 |
| 数据库设计 | 1 题 | 15 分 | E-R 图、关系模式、SQL |
| UML 建模 | 1 题 | 15 分 | 用例图、类图、时序图 |
| 算法与数据结构 | 1 题 | 15 分 | 算法填空、复杂度分析 |
| 设计模式 | 1 题 | 15 分 | 模式识别、代码填空 |
| C/C++/Java 编程 | 1 题（选做） | 15 分 | 代码实现、调试 |

**注意**：第 6 题为选做题，考生可从 C++ 和 Java 中任选其一

### 3.2 各题型详解

#### 1. 数据流图设计（15 分）

**考查内容**：
- 识别外部实体、加工、数据存储、数据流
- 补充缺失的数据流
- 识别错误的数据流
- 绘制顶层/0 层/1 层 DFD

**解题技巧**：
1. 先读题干，理解业务流程
2. 识别外部实体（与系统交互的人/组织）
3. 识别数据存储（数据库、文件）
4. 对照题干描述，找出缺失数据流
5. 注意数据平衡原则（输入=输出）

**常见考点**：
- 父子图平衡
- 数据流命名规范
- 加工命名（动词 + 名词）

#### 2. 数据库设计（15 分）

**考查内容**：
- E-R 图补充（实体、关系、属性）
- 关系模式转换
- 主键、外键识别
- 规范化（范式判断）
- SQL 语句编写

**解题技巧**：
1. 理解业务需求，识别实体
2. 确定实体间关系（1:1、1:n、m:n）
3. 转换为关系模式
4. 判断范式级别
5. 编写 SQL 查询语句

**常见考点**：
- E-R 图转关系模式
- 第三范式判断
- SQL 查询（连接、分组、排序）

#### 3. UML 建模（15 分）

**考查内容**：
- 用例图（参与者、用例、关系）
- 类图（类、属性、方法、关系）
- 时序图（对象、消息、生命线）
- 状态图、活动图

**解题技巧**：
1. 识别参与者（谁使用系统）
2. 识别用例（系统功能）
3. 确定关系（关联、继承、包含、扩展）
4. 补充缺失的类/关系
5. 识别设计模式

**常见考点**：
- 用例关系识别（include/extend）
- 类关系识别（继承、关联、聚合、组合）
- 设计模式识别

#### 4. 算法与数据结构（15 分）

**考查内容**：
- 算法填空（补全代码）
- 算法复杂度分析
- 数据结构应用
- 算法设计与实现

**解题技巧**：
1. 理解算法功能
2. 分析算法思路
3. 补全关键代码
4. 计算时间/空间复杂度

**常见考点**：
- 排序算法
- 查找算法
- 递归算法
- 动态规划
- 贪心算法

#### 5. 设计模式（15 分）

**考查内容**：
- 模式识别（根据描述判断模式）
- 模式应用（补充代码）
- 模式优缺点
- 模式对比

**解题技巧**：
1. 理解题目场景
2. 识别模式特征
3. 补充类图/代码
4. 说明模式作用

**常见考点**：
- 创建型：工厂、单例、建造者、原型
- 结构型：适配器、装饰器、代理、外观
- 行为型：观察者、策略、模板方法、命令

#### 6. 程序设计（选做，15 分）

**考查内容**（C++ 或 Java 二选一）：
- 代码填空
- 代码调试
- 类设计与实现
- 异常处理

**解题技巧**：
1. 理解程序功能
2. 分析代码结构
3. 补全关键代码
4. 注意语法规范

## 四、备考建议

### 4.1 备考时间规划

| 阶段 | 时间 | 内容 | 目标 |
|------|------|------|------|

| **冲刺阶段** | 2-3 周 | 真题模拟、查漏补缺 | 适应考试、提升分数 |

### 4.3 真题使用建议

|------|---------|------|

**建议刷题量**：
